This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это за проект

Личный проект ремонта квартиры в Краснодаре («Квартира Мурата Ахеджака 26»). Два слоя:

1. **Документация** в корневых папках на русском Markdown — это рабочий журнал решений, бюджета и планов.
2. **`webapp/`** — статический React + Vite сайт-витрина, отображающий бюджет, рендеры и планы. Хостится на Netlify.

Репозиторий **не** под git — изменения нужно фиксировать аккуратно вручную.

## Команды

Все команды веб-приложения запускаются из `webapp/`:

```
npm run dev       # локальный Vite dev-сервер
npm run build     # сборка в webapp/dist (Netlify публикует именно её)
npm run preview   # предпросмотр готовой сборки
```

Нет lint, нет тестов, нет typecheck — стек намеренно минималистичный (Vite + React 18, JS, без TypeScript).

## Архитектура

### Документация (источники истины для контента)

- `docs/project-goal.md` — цели и принципы проекта.
- `budget/budget.md` — описательная смета с пояснениями по каждой позиции и обоснованием цен (привязка к рынку Краснодара на конкретные даты).
- `decisions/decision-log.md` — журнал утверждённых/отменённых решений по шаблонной таблице.
- `decisions/compatibility-checks.md` — чеклист проверок совместимости перед закупкой/монтажом.
- `smart/smart-home-notes.md` — план автоматизации (Aqara/Zigbee, шторы, замок, хаб).
- `plans/*.pdf` — обмерный план и расстановка мебели.
- `renders/{living-room,bedroom,bathroom}/` — интерьерные визуализации по комнатам.

Workflow из README: новые планы → `plans/`, визуализации → `renders/`, расходы → `budget/budget.md`, решения → `decisions/decision-log.md`, перед закупкой сверяться с `decisions/compatibility-checks.md`.

### Webapp (`webapp/`)

- **Весь UI и весь контент в одном файле** — `webapp/src/App.jsx` (~870 строк). Массивы `summary`, `items`, `renders`, `plans` захардкожены константами в начале файла; ниже — компоненты страниц.
- **Роутинг свой, без библиотек**: `useState` + `window.history.pushState` + `popstate`. Три пути: `/` (бюджет), `/renders`, `/plans`. Helper `getPath()` нормализует и валидирует путь.
- **SPA-fallback на Netlify**: `webapp/public/_redirects` содержит `/* /index.html 200`. Без него прямые ссылки на `/renders` и `/plans` ломаются. `webapp/netlify.toml` указывает publish-директорию `dist`.
- **Привязка позиций бюджета к комнатам** делается функцией `getRoomsForBudgetItem(category, item)` (App.jsx:246) — это эвристика по подстрокам в названии. При добавлении новых позиций бюджета проверяй, что эта функция корректно отнесёт их к комнате, иначе фильтр «по комнатам» работать не будет.
- Стили — один глобальный `webapp/src/styles.css`, без CSS-модулей и без UI-библиотек.

### Дублирование ассетов (важно)

PDF и изображения существуют в **двух** местах одновременно:

- `plans/*.pdf`, `renders/<room>/*.jpg` — оригиналы для документации.
- `webapp/public/plans/*.pdf`, `webapp/public/renders/<room>/*.jpg` — копии, которые сервит Vite/Netlify.

При добавлении или замене файла обнови **обе** копии, иначе сайт и документация разойдутся. Пути в `App.jsx` (массивы `renders` и `plans`) указывают на `/renders/...` и `/plans/...`, то есть резолвятся из `webapp/public/`.

### Синхронизация бюджета

`budget/budget.md` — повествовательный источник, `App.jsx` массив `items` — то, что видит пользователь. При изменении смет синхронизируй обе стороны: сначала обнови `budget.md` (с пояснением и датой ориентира цены), потом перенеси изменения в `items` в `App.jsx`. Сводные значения в массиве `summary` (App.jsx:3) должны соответствовать таблице «Сводка по позициям из рендеров» в `budget.md`.

## Стиль контента

Вся пользовательская копия и документация — на русском. Цены в рублях, в `App.jsx` форматируются через `Intl.NumberFormat("ru-RU")`. Числительные склоняются хелпером `pluralize()` (App.jsx:233).
